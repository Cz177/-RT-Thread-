# 智能元器件收纳管家 - 项目技术文档

---

## 一、项目概述

### 1.1 项目名称

智能元器件收纳管家

### 1.2 项目简介

基于RT-Thread操作系统的智能电子元件存储管理系统，通过LVGL图形界面实现器件的添加、查找和取件功能，支持电机精确定位和舵机取件门控制。

### 1.3 功能特性

| 功能模块 | 功能描述 | 实现方式 |
| --- | --- | --- |
| 器件管理 | 添加、查找、删除电子元件 | FlashDB键值存储 |
| 位置定位 | 电机精确定位到指定位置 | PID算法控制 |
| 取件控制 | 控制舵机打开/关闭取件门 | PWM信号控制 |
| 可视化界面 | 触摸屏交互操作 | LVGL 8.3.11 |
| 数据持久化 | 器件信息断电保存 | FlashDB |

## 六、工作流程图

### 6.1 系统启动流程

```mermaid
sequenceDiagram
    participant Main as main()
    participant Event as global_event
    participant DB as FlashDB
    participant Motor as 电机线程
    participant Servo as 舵机线程
    participant GUI as LVGL界面

    Main->>Event: 创建全局事件标志
    Main->>DB: 初始化数据库
    Main->>Motor: 初始化电机线程(含归零)
    Main->>Servo: 初始化舵机线程
    Main->>GUI: 加载主界面
    Note over Main: 系统进入主循环
```

### 6.2 添加器件流程

```mermaid
flowchart TD
    A[用户点击添加按钮] --> B[获取Tab类型]
    B --> C{电阻/电容/电感?}
    C -->|电阻| D[读取电阻参数]
    C -->|电容| E[读取电容参数]
    C -->|电感| F[读取电感参数]
    D --> G[组装JSON数据]
    E --> G
    F --> G
    G --> H[components_add_kv]
    H --> I{检查重复?}
    I -->|是| J[弹出提示]
    I -->|否| K[分配存储位置]
    K --> L[发送位置到电机队列]
    L --> M[发送层数到舵机队列]
    M --> N[等待用户完成放置]
    N --> O[更新数据库]
```

### 6.3 取件流程

```mermaid
flowchart TD
    A[用户选择器件] --> B[输入数量]
    B --> C[event_get_component_cb]
    C --> D{添加还是取出?}
    D -->|添加| E[更新数量+]
    D -->|取出| F{数量是否足够?}
    F -->|否| G[弹出警告]
    F -->|是| H[更新数量-]
    H --> I{是否取完?}
    I -->|是| J[删除KV记录]
    I -->|否| K[更新KV记录]
    E --> L[发送位置到电机MQ]
    J --> L
    K --> L
    L --> M[发送层数到舵机MQ]
    M --> N[电机PID定位]
    N --> O[发送EVENT_POSITION_OK]
    O --> P[舵机开门]
    P --> Q[等待用户取件]
    Q --> R[发送EVENT_PICK_FINNISH]
    R --> S[舵机关门]
```

### 6.4 电机PID控制流程

```mermaid
flowchart TD
    A[接收目标位置] --> B[计算目标脉冲数 target=pos*290]
    B --> C[读取编码器计数]
    C --> D[计算实际位置 actual]
    D --> E[计算误差 error=target-actual]
    E --> F{误差<5且连续10次?}
    F -->|是| G[停止电机]
    G --> H[发送EVENT_POSITION_OK]
    F -->|否| I[PID计算 pwm=Kp*e+Ki*∫e+Kd*de]
    I --> J[限幅处理 -30~30]
    J --> K[设置PWM输出]
    K --> L[延时20ms]
    L --> C
```

## 七、通信协议表

### 7.1 线程间通信

| 通信方式 | 名称  | 数据类型 | 用途  |
| --- | --- | --- | --- |
| 消息队列 | motor_mq | uint8_t | 传递目标位置(0-11) |
| 消息队列 | servo_mq | uint8_t | 传递目标层数(0-2) |
| 事件标志 | global_event | rt_uint32_t | 线程同步信号 |

### 7.2 事件标志定义

| 事件名称 | 位掩码 | 触发时机 |
| --- | --- | --- |
| EVENT_PICK_FINNISH | 1 << 2 | 用户完成取件操作 |
| EVENT_POSITION_OK | 1 << 3 | 电机定位完成 |
| EVENT_TOUCH_SCREEN | 1 << 4 | 触摸屏被触摸 |
| EVENT_ZERO_OK | 1 << 5 | 电机归零完成 |

### 7.3 JSON数据格式

**器件信息JSON格式：**

| 字段名 | 类型  | 含义  | 示例  |
| --- | --- | --- | --- |
| type | string | 器件类型 | "Resistor"/"Capacitor"/"Inductor" |
| val | string | 器件值 | "10k"/"100n"/"10u" |
| pkg | string | 封装类型 | "0603"/"0805"/"1206" |
| rated val | string | 额定值 | "1/4W"/"6.3V"/"1A" |
| accuracy | string | 精度/容差 | "5%"/"10%"/"J" |
| other | string | 其他信息 | "贴片"/"直插" |

**示例：**

```json
{
    "type": "Resistor",
    "val": "10k",
    "pkg": "0603",
    "rated val": "1/4W",
    "accuracy": "5%",
    "other": "贴片"
}
```

| 参数  | 值   | 说明  |
| --- | --- | --- |
| Kp  | 0.35 | 比例系数 |
| Ki  | 0.05 | 积分系数 |
| Kd  | 0   | 微分系数 |
| 控制周期 | 20ms | PID计算间隔 |

### 10.2 位置映射

| 位置编号 | 位置索引(pos) | 层数(layer) | 脉冲数(target) |
| --- | --- | --- | --- |

| 12-23 | 0-11 | 1   | 0-3190 |
| 24-35 | 0-11 | 2   | 0-3190 |

**计算公式：** `target = pos × 290`
